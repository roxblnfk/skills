# Built-in trusted-vendors list for `llm/skills`.
#
# A package whose Composer name matches any of these patterns is allowed to
# ship AI skills into a consumer project without an explicit opt-in. The goal
# is purely safety: AI skills are Markdown instructions executed by an agent,
# so a malicious package could ship a prompt-injection payload. The trust
# list narrows the attack surface to maintainers we are willing to vouch for.
#
# Pattern syntax (see \LLM\Skills\Config\VendorPattern):
#   - `vendor/*`       — every package in the vendor namespace
#   - `vendor/package` — exact package match
#
# Maintainers extend this list by PR. End users override locally via:
#
#     "extra": {
#       "skills": {
#         "trusted": ["my-org/*", "acme/skills-pro"],
#         "trusted-replace": true
#       }
#     }
#
# Format: one pattern per line. Blank lines and lines starting with `#` are
# ignored. Surrounding whitespace is trimmed.

doctrine/*
internal/*
laravel/*
llm/*
moonshine/*
spiral/*
